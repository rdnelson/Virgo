The breakpoints window

This window helps you view and manage your breakpoints. I assume you know what a breakpoint is. The breakpoints window consists of two tabs:

Current breakpoints tab:

This tab shows you the list of all current breakpoints. Each breakpoint is expressed as a meaningful sentence.

The Remove button will remove all the breakpoints that are selected. The currently selected breakpoints are highlighted in blue. You can multi-select any desired combination of breakpoints to remove them.

The checkbox at the bottom of the tab reads Break points are enabled. It is checked by default. You can uncheck it if you want to disable all breakpoints temporarily, without having to delete them and then re-add them. This option comes in handy when you want to try to run the whole program once or twice, but still want to debug it again after that.

Add tab:

This tab allows you to add any kind of breakpoint.

To add a code breakpoint, fill out the first sentence. The memory address you specify must be a valid hexadecimal word. Press Add when you're done. A more convenient way to add a code breakpoint is to check the checkbox beside the desired instruction in the code window.

To add a register breakpoint, fill out the second sentence. The combo box provided contains the list of registers to choose from. Choose the break condition from the next combo box. Next fill out the comparison value (if needed) and press Add. The comparison value must be a valid signed decimal entry.

To add a memory or port breakpoint, fill out the third sentence. Select byte or word from the first combo box; and memory or port from the second. Next fill out the desired address (hexadecimal word), choose the appropriate break condition, and specify the comparison value if needed (signed decimal). Press Add when you're done.

When you add a breakpoint, you are automatically taken to the Current breakpoints tab, and the newly added breakpoint is automatically highlighted.
