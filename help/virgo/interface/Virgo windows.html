Virgo windows

Virgo provides multiple windows, each for managing a separate part of the program. The main window must always be active, while all other windows can be enabled/disabled as convenient. All windows are enabled by default, except the memory window which consumes a big part of the system resources and should be enabled only when running Virgo on a powerful machine.

When you first load Virgo, only the main window appears and most options are disabled. After you load an OBJ file, other windows appear, and they are automatically lined up in a neat way. You can customize the number, location, and size of the your windows. If you want to restore the default appearance, choose Arrange neatly from the Windows menu. You can also choose Close all from the Windows menu to close all windows (except the main window) at once.

Below is a brief description of every window:
 

The main window

Contains all the features you need to manage OBJ files and run/debug your code. It also contains a black area that serves as a simulated P86 screen.
More details.
The registers and ports window
Contains fields that enable you to view all P86 registers, flags, and ports. In this window, you can modify the contents of registers, flags, but you can't mess with ports.
More details.
The code window
Contains all the information about your code, including the address of every instruction in memory, labels given to certain instruction, and the assembly source for every instruction. There is a checkbox beside every instruction, that serves as a breakpoint indicator.
More details.
The memory window
Shows you the contents of every memory cell in the P86 memory block. You can modify the contents of any cell by double clicking on it. This window is not open by default. Read the first paragraph in this documemt for details.
More details.
The watches window
Shows you a list of all watches and their current values. You can use this window to add/remove any kind of watches.
More details.
The breakpoints window
Shows you a list of all current breakpoints. You can use this window to add/remove any kind of breakpoint.
More details.
