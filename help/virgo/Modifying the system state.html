Modifying the system state

Virgo simulates a simple machine (called P86) and shows you how your code modifies the state of that machine. Ideally, the only way to change the state of that machine is through the instructions in your code. However, to help you experiment with the computer machine, and get the feel of how it behaves, Virgo allows you to manually change the state of the machine (eg. registers, flags, memory)

Virgo only allows you to modify things while your program is idle. That is, you can't manually change anything while your program your running. If you want to, set a breakpoint (or press pause at the right split second), modify as needed, then run again.

If you end up modifying the state too much!, you can always press the reset button in the main window. This will reload your instructions and reset the system to its initial state.
 

Modifying registers:

In the registers and ports window, every register is represented by a button that carries its value. Press on a button to change the value of a specific register. The new value has to be a valid hexadecimal word (4 hex digits) or else the new entry won't be accepted. If you try to change the value of IP, the new value has to correspond to the starting address of some instruction of your code, otherwise it will be considered invalid. Note that changing the FLAGS register might automatically change the system flags as well.

Modifying flags:

In the registers and ports window, flags are represented by check boxes. If a flag is checked, it means that it is set, i.e equal to one. click on any flag to set/reset its state. Note that modifying a flag will automatically change the value of the FLAGS register.

Modifying ports:

Virgo does not allow you to modify the values of ports manually, mainly because ports are not persistent containers. That is, unlike memory cells, ports are not there to hold information, but to interact with the devices connected to the computer system. The only way to read/write a port is using the IN/OUT instructions respectively.

Modifying memory cells:

In the memory window, every memory location is represented as a separate cell. Double click on any cell to change its value. The new value has to be a valid hexadecimal byte or else it won't be accepted. You can't change the value of a word (2 bytes) at once. To do that, change the value of every byte separately (little endian!). Also note that if you modify the part of memory that contains your byte code, I can GUARANTEE you that your code won't work any more!
