Debugging your program

Virgo provides multiple running modes that help you debug your program, ranging from executing one instruction at a time, to running the whole program at once.

Single step:

Executes a single assembly instruction (the current instruction) and immediately modifies the state of the machine. This mode is useful when you have pin-pointed the troublesome spot in your code and want to take a closer look.
Step over:
Behaves exactly like single step. Except that if the current instruction is a CALL or INT, the whole subroutine that is called is executed. This mode is useful if you're single stepping and you come across a CALL that you know is not the cause of any problems (eg. CALL displayText or INT 4).
Run (or play):
Sequentially executes the instructions in your code, starting at the current instruction. Running will stop when any of the following happens:
  • The end of code is reached (HLT instruction)
  • A breakpoint is reached
  • An invalid instruction is reached
  • IP is set to a value that is not the start of an instruction
Run until selected code line:
Behaves exactly like run, except that execution will also stop when the currently selected code line is reached. You can think of this mode as a Run with a temporary breakpoint.
Run until CALL:
Behaves exactly like run, except that execution will also stop when the first CALL instruction is reached.
To make debugging even more efficient. Make use of Virgo's breakpoints and watches.
 
 

Note:

The current instruction is the instruction that IP currently points to. In the code window, you will find this instruction highlighted in red. To change the current instruction, you can change the value of IP in the registers and ports window.
