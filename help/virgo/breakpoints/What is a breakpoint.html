What is a breakpoint?

A breakpoint is a condition that, when evaluates to true, causes the debugger to stop executing your code. This results in the sequential flow of your instructions being broken. Hence the name breakpoint.

Breakpoints are very useful when you need to find out WHEN a specific event occurs.

For example, let's say that you need to know if a specific region of code causes AX to contain the value zero. What you do is you set a breakpoint at the end of that code block, and another breakpoint that fires when AX is equal to zero. Now you can run your code and see if the latter breakpoint occurs before the first.
 

Types of breakpoints

Virgo allows for a wide range of breakpoint conditions:
 

Code breakpoints

break when a certain instruction is reached. you can manage that kind of breakpoints from the breakpoints window, or simply by clicking on the checkbox beside the desired instruction in the code window.

Memory breakpoints

break when a specific memory byte or word is:
  • modified
  • equal to a value
  • less than a value
  • greater than a value
  • less than or equal to a value
  • greater than or equal to a value
  • not equal to a value
    • (Note that all comparisons are based on signed 2's complement interpertation)

    Port breakpoints

    The same features as above, applied to 8-bit and 16-bit ports.

    Register breakpoints

    The same features as above, applied to all 16-bit registers.


    The breakpoints window is a separate window that shows you the list of current breakpoints and allows you to modify that list.
