Managing breakpoints

Adding/removing a code breakpoint:

  • Find the desired instruction in the code window
  • Double click on the check box beside it to check it or uncheck it. The state of the check box determines whether this instruction is a breakpoints or not.
Alternatively:
  • Select the Add tab in the breakpoints window. This tab contains a form with all the details you need to specify for a breakpoint, laid out in the form of sentences
  • Fill out the first sentence, specifying the address of the start of the desired instruction (as a hexadecimal word)
  • Press Add

Adding a register breakpoint:

  • Select the Add tab in the breakpoints window. This tab contains a form with all the details you need to specify for a watch, laid out in the form of sentences
  • Fill out the second sentence, specifying the register you want to break on, the condition you want to break on, and the (signed decimal) value of comparison if necessary
  • Press Add

Adding a memory or port breakpoint:

  • Select the Add tab in the breakpoints window. This tab contains a form with all the details you need to specify for a watch, laid out in the form of sentences
  • Fill out the third sentence, specifying whether you want to break on a port or register, whether this port/register is a byte or word, the address of the start of that byte/word, the condition upon which to break, and the (signed decimal) value if necessary
  • Press Add

Removing breakpoints:

  • Select the Current breakpoints tab in the breakpoints window. This tab contains a list of all the current breakpoints.
  • Select one or more breakpoints from the list. You can multi-select by holding down Shift or Ctrl on WIN NT platforms. The selected breakpoints are highlighted in blue. Once you're satisfied with you selection, click Remove and they're gone.

Note:

You can only add/remove breakpoints when your program is idle (i.e. not running).
