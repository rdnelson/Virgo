Error messages

Here is a list of the error messages that Virgo might decide to shoot at you for one reason or another. The messages are arranged by the task you were trying to perform when they occured. Every message has its explanation beside it, along with some suggestions about what might be the problem:

When opening OBJ files:

filename does not exist or is unreachable
The file you wanted to open was not found. Either you typed the name of a file that does not exist (check for spelling errors), or you do not have read access to the file that you specified (consult network administrator).
filename is corrupted
The file that you tried to open in not a valid OBJ file. It contains bad data inside it. Try re-assembling your original ASM file.

When debugging your program:

End of code is already reached
The last time you executed the program, a HLT instruction was reached. This is the end of the program and you can't run your code any further! If you want to run your code from the start again, press Reset in the main window.


Instruction set IP to a location that is not the start of an instruction

After executing an instruction, IP ended up pointing to a location that does not correspond to the start of any insturction in your code. This means something went wrong for sure! I suggest you look for the following pitfalls:
  • You ended your code but forgot to add a HLT instruction at the end
  • Your PUSH/POP pairs do not match (especially during subroutine CALLs)
  • You did not set up the vector table CORRECTLY prior to calling a software interrupt (using INT)
The first byte in the instruction does not confirm to any known instruction
or
Bad REG field ....
or
Instruction contains an invalid address mode
One of the instructions does not follow the standard instruction-encoding format. Chances are, in some previous instructions you have modified some memory locations that contain your code. Make sure that all instructions that modify memory modify the RIGHT locations.

When modifying the system state:

x is not a hexadecimal byte/word!
You are trying to modify a register or a memory cell, but the new value you enter is not valid. A register must contain a valid HEXADECIMAL word. A memory address must contain a valid HEXADECIMAL byte. If you don't understand the meaning of those sentences, ask your instructor.
New IP does not point to the start of an instruction!
You are trying to modify register IP. This register contains the memory address of the next instruction to be executed. The new value that you want IP to contain does not confirm to the start of any instruction in memory. Look at the table in the code window to see the start address of every instruction in your code.
